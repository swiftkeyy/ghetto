## 🔧 Технические детали

### SSH Manager возможности:

#### WireGuard:
- ✅ Автоматическая генерация ключей на VPS
- ✅ Добавление peers в wg0.conf
- ✅ Применение изменений без перезагрузки
- ✅ Автоматическое назначение IP адресов
- ✅ Удаление неактивных peers

#### Xray (VLESS):
- ✅ Генерация UUID на VPS
- ✅ Добавление клиентов в config.json
- ✅ Перезапуск сервиса
- ✅ Создание VLESS URI
- ✅ Удаление клиентов

#### Мониторинг:
- ✅ CPU usage
- ✅ RAM usage
- ✅ Disk usage
- ✅ Network traffic (RX/TX)
- ✅ Количество подключенных пользователей
- ✅ Health check (ping, service status)

## 🚀 Как использовать

### 1. Настройте VPS сервер

Следуйте инструкциям в **SSH_VPS_SETUP.md**:

```bash
# Установите WireGuard или Xray
# Настройте SSH доступ (ключ или пароль)
# Откройте нужные порты в firewall
```

### 2. Добавьте сервер через бота

```
1. Откройте бота
2. ⚡ Админ-панель
3. 🌍 Серверы
4. ➕ Добавить сервер
5. Заполните данные:
   - Название, IP, порт
   - SSH credentials (ключ/пароль)
   - Настройки протокола
```

### 3. Протестируйте

```
1. Серверы → Выберите сервер
2. 🔄 Проверить
3. Убедитесь что health check проходит
```

### 4. Готово!

Теперь когда пользователь:
- Выбирает сервер и жмёт "Подключиться"
- Бот автоматически создаёт ключи на VPS через SSH
- Пользователь получает QR-код и готовый конфиг
- При удалении устройства - ключи удаляются с VPS

**Всё автоматически!** 🎉

## 🔐 Безопасность

### SSH Credentials:

**Хранятся в:**
- Database (server.config JSON field) - зашифровано
- Или в переменных окружения

**Рекомендации:**
1. Используйте SSH ключи вместо паролей
2. Создайте отдельного пользователя для бота
3. Ограничьте sudo права только нужными командами
4. Используйте fail2ban для защиты SSH
5. Регулярно ротируйте ключи

## 🐛 Known Issues & Limitations

### Текущие ограничения:
- SSH подключения синхронные (могут быть медленными при большой нагрузке)
- Поддержка только WireGuard и VLESS (VMess/Shadowsocks - локально)
- Health checks каждые 5 минут (настраиваемо)

### Планы на будущее:
- Асинхронный пул SSH подключений
- Поддержка больше протоколов через SSH
- Автоматическая миграция между серверами
- Балансировка нагрузки
- A/B тестирование серверов
